PARTE 1 — EXAMEN ESCRITO Sin ordenador · ~1 hora · 4 puntos
Escenario: GGZone — Plataforma de Torneos de E-Sports

GGZone es una plataforma online donde se organizan torneos de videojuegos para jugadores aficionados.

En la plataforma se registran jugadores con su alias (nombre de usuario), nombre real, ciudad de residencia y una puntuación de rango numérica. Cada jugador puede inscribirse en varios torneos, y en cada torneo pueden participar muchos jugadores.

Los torneos están siempre asociados a un videojuego concreto (por ejemplo: Minecraft, Counter-Strike 2 o Clash Royale); un mismo juego puede tener varios torneos a lo largo del tiempo. Cada torneo tiene un nombre, una fecha de celebración, un premio máximo en euros y un número máximo de participantes.

Cuando un jugador se inscribe en un torneo, se registra la fecha de inscripción. Una vez finalizado el torneo, también se guarda su posición final.

1b) [0,5 pts] Justificación de las claves primarias.
✔ Respuesta esperada

Cada tabla usa un campo Autonumérico como PK. Se elige porque:

  • Es único para cada registro (lo genera Access automáticamente).
  • No tiene significado de negocio, por lo que nunca cambia.
  • Permite relacionar tablas de forma eficiente mediante claves foráneas.
1c) [1,0 pts] Claves foráneas, relaciones y cardinalidades.
✔ Relaciones del modelo
┌──────────┐ 1 N ┌──────────────────┐ │ JUEGOS │───────────────────────────│ TORNEOS │ │ IDJuego │◄── IDJuego (FK) ───────────│ IDTorneo (PK) │ └──────────┘ └────────┬─────────┘ │ 1 │ N ┌────────────┐ 1 N ┌────────────────────┐ │ JUGADORES │──────────────────────│ INSCRIPCIONES │ │ IDJugador │◄─ IDJugador (FK) ────│ IDInscripcion (PK) │ └────────────┘ │ IDJugador (FK) │ │ IDTorneo (FK) ────┘ │ FechaInscripcion │ └────────────────────┘
  • TORNEOS.IDJuego → JUEGOS.IDJuego 1 : N — Un juego puede tener varios torneos.
  • INSCRIPCIONES.IDJugador → JUGADORES.IDJugador 1 : N — Un jugador puede tener varias inscripciones.
  • INSCRIPCIONES.IDTorneo → TORNEOS.IDTorneo 1 : N — Un torneo puede tener varios jugadores inscritos.
La relación JUGADORES ↔ TORNEOS es N:M. Se resuelve con la tabla intermedia INSCRIPCIONES, que contiene las dos FKs y los datos propios de la inscripción.
