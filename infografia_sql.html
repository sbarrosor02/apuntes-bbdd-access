SELECT WHERE ORDER BY GROUP BY JOIN Funciones
SELECT Seleccionar columnas 01
SELECT * FROM Tabla; SELECT col1, col2 FROM Tabla; SELECT DISTINCT col FROM Tabla; SELECT col AS Alias FROM Tabla;
  • * → todas las columnas
  • DISTINCT → elimina duplicados
  • AS → renombra columna (alias)
WHERE Filtrar filas 02
SELECT * FROM T WHERE col = 'valor' AND edad > 18 OR pais = 'ES';
= <> < > <= >= AND OR NOT

ORDER BY Ordenar resultados 04
SELECT * FROM T ORDER BY col1 ASC; -- ascendente ORDER BY col1 DESC; -- descendente ORDER BY col1, col2 ASC;
  • ASC → de menor a mayor (defecto)
  • DESC → de mayor a menor
  • Se pueden ordenar por varias columnas
GROUP BY + HAVING Agrupar datos 05
SELECT col, COUNT(*) AS total FROM Tabla GROUP BY col HAVING COUNT(*) > 2;
  • GROUP BY → agrupa filas iguales
  • HAVING → filtra grupos (≠ WHERE)
  • Columnas en SELECT deben estar en GROUP BY o en función

JOIN Combinar tablas relacionadas 07
SELECT A.col1, B.col2 FROM TablaA A INNER JOIN TablaB B ON A.ID = B.IDForeign;
  • INNER JOIN → solo filas coincidentes en ambas tablas
  • ON → condición de unión (PK = FK)
  • Usar alias (A, B) para abreviar nombres
Tipos de JOIN INNER JOIN — solo coincidencias
LEFT JOIN — todas las de la izq. + coincidencias
RIGHT JOIN — todas las de la der. + coincidencias

Estructura completa de una consulta SELECT Orden obligatorio de las cláusulas 09
SELECT columnas / funciones FROM tabla(s) / JOIN WHERE condición de filas GROUP BY columna(s) HAVING condición de grupos ORDER BY columna [ASC|DESC];
SELECT → qué mostrar
FROM → de dónde obtener datos
WHERE → filtro ANTES de agrupar
GROUP BY → agrupa filas iguales
HAVING → filtro DESPUÉS de agrupar
ORDER BY → ordena el resultado final
Ejemplo real Consulta completa 10
SELECT Pais, COUNT(*) AS Jugadores, AVG(Puntuacion) AS Media FROM Jugadores WHERE CuentaActiva = True GROUP BY Pais HAVING COUNT(*) >= 2 ORDER BY Media DESC;

Subconsultas SELECT dentro de SELECT 13
SELECT * FROM T WHERE col = ( SELECT MAX(col) FROM T ); WHERE ID IN ( SELECT ID FROM Otra );
WHERE vs HAVING ¿Cuándo usar cada uno? 14
WHERE
  • Filtra filas individuales
  • Se ejecuta antes del GROUP BY
  • No puede usar funciones agregadas
  • Ejemplo: WHERE edad > 18
HAVING
  • Filtra grupos ya formados
  • Se ejecuta después del GROUP BY
  • Sí puede usar funciones agregadas
  • Ejemplo: HAVING COUNT(*) > 2

Claves PK y FK Integridad referencial 16
  • PRIMARY KEY (PK) → identifica cada fila de forma única, no puede ser NULL ni repetirse
  • FOREIGN KEY (FK) → referencia a la PK de otra tabla, crea la relación
  • Integridad referencial → no puedes insertar un FK que no exista como PK
