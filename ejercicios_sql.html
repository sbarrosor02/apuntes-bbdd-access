Estructura de una consulta SQL

SELECT columnas
Qué campos mostrar. Usa * para todos.
FROM tabla
De qué tabla sacar los datos.
WHERE condición
Filtrar filas. Usa =, >, <, LIKE.
ORDER BY campo
Ordenar resultados. Añade DESC para invertir.
GROUP BY campo
Agrupar filas para usar funciones de agregado.
COUNT(*)
Contar filas de cada grupo.
SUM / AVG
Sumar o calcular la media de un campo numérico.
AND / OR
Combinar varias condiciones en el WHERE.
1️⃣

Bloque 1 — SELECT básico

Aprende a seleccionar filas y columnas de una tabla

concepto

La cláusula SELECT

Con SELECT elegimos qué columnas queremos ver, y con FROM indicamos de qué tabla vienen los datos.

  • SELECT * FROM JUGADORES — muestra todas las columnas.
  • SELECT NombreUsuario, Pais FROM JUGADORES — muestra solo esas dos columnas.
1
Mostrar todos los jugadores Fácil

Escribe una consulta que muestre todas las columnas y todos los registros de la tabla JUGADORES.

SQL In [1]:
Usa el asterisco * para seleccionar todas las columnas. La tabla se llama exactamente JUGADORES.
Solución
SELECT *
FROM JUGADORES
2
Solo nombre de usuario y país Fácil

Muestra únicamente los campos NombreUsuario y Pais de todos los jugadores. Solo dos columnas, sin el resto.

SQL In [2]:
Escribe los nombres de los campos separados por comas después de SELECT. Respeta las mayúsculas de los nombres de campo.
Solución
SELECT NombreUsuario, Pais
FROM JUGADORES
3
Nombre y velocidad de las naves Fácil

Muestra el NombreNave, el Modelo y la VelocidadMax de todas las naves de la tabla NAVES.

SQL In [3]:
Son tres campos separados por comas. La tabla es NAVES.
Solución
SELECT NombreNave, Modelo, VelocidadMax
FROM NAVES
2️⃣

Bloque 2 — WHERE: filtrar filas

Selecciona solo los registros que cumplan una condición

concepto

La cláusula WHERE

WHERE nos permite filtrar registros, quedándonos solo con los que cumplen una condición.

  • Texto: WHERE Pais = 'España' — las comillas simples son obligatorias para texto.
  • Números: WHERE VelocidadMax > 500 — sin comillas.
  • Sí/No: WHERE CuentaActiva = true
  • Múltiples condiciones: WHERE Pais = 'España' AND CuentaActiva = true
4
Jugadores de España Fácil

Muestra todos los campos de los jugadores cuyo Pais sea 'España'.

SQL In [4]:
La condición de texto necesita comillas simples: Pais = 'España'. Resultado esperado: 2 filas.
Solución
SELECT *
FROM JUGADORES
WHERE Pais = 'España'
5
Naves muy rápidas Fácil

Muestra el NombreNave y la VelocidadMax de todas las naves cuya velocidad máxima sea mayor de 500.

SQL In [5]:
Los números no llevan comillas: VelocidadMax > 500. Resultado esperado: 4 filas.
Solución
SELECT NombreNave, VelocidadMax
FROM NAVES
WHERE VelocidadMax > 500
6
Solo cuentas activas Fácil

Muestra el NombreUsuario y Email de los jugadores que tienen CuentaActiva = true.

SQL In [6]:
Los campos Sí/No se comparan con true o false sin comillas. Resultado esperado: 6 filas.
Solución
SELECT NombreUsuario, Email
FROM JUGADORES
WHERE CuentaActiva = true
7
Circuitos difíciles Medio

Muestra el NombreCircuito, el Planeta y la LongitudKm de los circuitos cuyo CodigoDificultad sea 'D' (Difícil). También muestra los de dificultad 'M' usando OR.

SQL In [7]:
Usa OR para combinar dos condiciones: ... = 'D' OR ... = 'M'. Resultado esperado: 3 filas.
Solución
SELECT NombreCircuito, Planeta, LongitudKm
FROM CIRCUITOS
WHERE CodigoDificultad = 'D' OR CodigoDificultad = 'M'
3️⃣

Bloque 3 — ORDER BY: ordenar resultados

Controla el orden en que aparecen los datos

concepto

La cláusula ORDER BY

ORDER BY se coloca al final de la consulta y ordena los resultados según el campo indicado.

  • ORDER BY NombreUsuario — orden ascendente (A → Z, 1 → 9). Es el orden por defecto.
  • ORDER BY VelocidadMax DESC — orden descendente (mayor → menor, Z → A).
  • Puedes combinar: ORDER BY Pais ASC, NombreUsuario ASC
8
Jugadores ordenados por país Fácil

Muestra NombreUsuario y Pais de todos los jugadores, ordenados alfabéticamente por país.

SQL In [8]:
ORDER BY Pais al final de la consulta ordena de A a Z por defecto.
Solución
SELECT NombreUsuario, Pais
FROM JUGADORES
ORDER BY Pais
9
Naves más rápidas primero Fácil

Muestra NombreNave, Modelo y VelocidadMax de todas las naves, ordenadas de mayor a menor velocidad.

SQL In [9]:
Para ordenar de mayor a menor añade DESC después del nombre del campo.
Solución
SELECT NombreNave, Modelo, VelocidadMax
FROM NAVES
ORDER BY VelocidadMax DESC
10
Ranking de puntos Medio

Muestra IDNave, Posicion y PuntosGanados de la tabla RESULTADOS. Muestra primero los que más puntos han ganado (orden descendente por PuntosGanados). En caso de empate, ordena por Posicion ascendente.

SQL In [10]:
Puedes ordenar por varios campos separándolos con comas: ORDER BY PuntosGanados DESC, Posicion ASC.
Solución
SELECT IDNave, Posicion, PuntosGanados
FROM RESULTADOS
ORDER BY PuntosGanados DESC, Posicion ASC
4️⃣

Bloque 4 — GROUP BY: agrupar y contar

Calcula totales, medias y conteos por grupos

concepto

GROUP BY y funciones de agregado

GROUP BY agrupa filas con el mismo valor en un campo, y luego podemos aplicar funciones sobre cada grupo:

  • COUNT(*) — cuántas filas hay en cada grupo.
  • SUM(campo) — suma de los valores del campo.
  • AVG(campo) — media de los valores.
  • MAX(campo) / MIN(campo) — valor máximo / mínimo.

Regla importante: todo campo del SELECT que no sea una función de agregado debe estar en el GROUP BY.

11
Naves por modelo Medio

Muestra cuántas naves hay de cada Modelo. El resultado debe tener dos columnas: Modelo y el conteo (puedes llamarlo TotalNaves con AS).

SQL In [11]:
Estructura: SELECT Modelo, COUNT(*) AS TotalNaves FROM NAVES GROUP BY Modelo. Resultado esperado: 4 filas (un Interceptor, Destroyer, Cruiser y Stealth).
Solución
SELECT Modelo, COUNT(*) AS TotalNaves
FROM NAVES
GROUP BY Modelo
12
Jugadores por país Medio

Cuenta cuántos jugadores hay de cada Pais. Muestra el país y el número de jugadores. Ordena el resultado de mayor a menor número de jugadores.

SQL In [12]:
Puedes combinar GROUP BY con ORDER BY. Ordena por el alias del COUNT: ORDER BY Total DESC.
Solución
SELECT Pais, COUNT(*) AS Total
FROM JUGADORES
GROUP BY Pais
ORDER BY Total DESC
13
Puntos totales por nave Medio

Calcula el total de puntos acumulados por cada nave en la tabla RESULTADOS. Muestra el IDNave y la suma de PuntosGanados. Ordena de mayor a menor puntuación total.

SQL In [13]:
Usa SUM(PuntosGanados) AS TotalPuntos en el SELECT. Agrupa por IDNave y ordena por TotalPuntos DESC.
Solución
SELECT IDNave, SUM(PuntosGanados) AS TotalPuntos
FROM RESULTADOS
GROUP BY IDNave
ORDER BY TotalPuntos DESC
5️⃣

Bloque 5 — Consultas combinadas

WHERE + ORDER BY + GROUP BY juntos

importante

⚠️ Orden obligatorio de las cláusulas: en SQL siempre debes escribirlas en este orden: SELECT → FROM → WHERE → GROUP BY → ORDER BY. Si los cambias de sitio obtendrás un error de sintaxis.

14
Jugadores activos de España ordenados Medio

Muestra el NombreUsuario y FechaRegistro de los jugadores de España con CuentaActiva = true. Ordénalos por FechaRegistro (los más recientes primero).

SQL In [14]:
Combina dos condiciones con AND: WHERE Pais = 'España' AND CuentaActiva = true. Luego ordena por FechaRegistro DESC.
Solución
SELECT NombreUsuario, FechaRegistro
FROM JUGADORES
WHERE Pais = 'España' AND CuentaActiva = true
ORDER BY FechaRegistro DESC
15
Media de velocidad por modelo (Interceptor y Destroyer) Difícil

Calcula la velocidad media de las naves agrupadas por Modelo, pero solo para los modelos 'Interceptor' y 'Destroyer'. Muestra el modelo y la media. Ordena de mayor a menor media.

SQL In [15]:
Primero filtra con WHERE para quedarte solo con esos dos modelos, luego agrupa con GROUP BY y usa AVG(VelocidadMax). Resultado esperado: 2 filas.
Solución
SELECT Modelo, AVG(VelocidadMax) AS MediaVelocidad
FROM NAVES
WHERE Modelo = 'Interceptor' OR Modelo = 'Destroyer'
GROUP BY Modelo
ORDER BY MediaVelocidad DESC
